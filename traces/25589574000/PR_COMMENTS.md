# AI Permissions Fix and Agent Configuration Restructure

This PR addresses the AI permissions issues referenced in https://github.com/ai4curation/aidocs/issues/39 and makes several improvements to the UBERON AI agent configuration.

## Changes Made

### 1. Security: Fixed Bash Permission Wildcard
**File**: `.claude/settings.json`

- **Before**: `"Bash(*)"`  - overly permissive wildcard allowing any Bash command
- **After**: `"Bash"` - standard permission without wildcards
- **Impact**: Tightens security while maintaining necessary Bash functionality for ontology editing tasks

### 2. Quality Assurance: Added Task Completion Verification
**File**: `.claude/settings.json` (new Stop hook)

Added an agent-based Stop hook that verifies task completion before allowing the agent to stop. The hook:
- Checks whether the agent actually completed its assigned task
- Validates that proper deliverables were created (commits, PRs, GitHub comments)
- Looks for evidence of actual tool use (gh commands, git push, etc.) rather than just text claims
- Prevents premature exits where work appears incomplete

This ensures the @dragon-ai-agent doesn't claim completion without actually finishing the task.

### 3. Structure: Migrated from Agents to Skills
**Changed files**:
- Moved: `.claude/agents/*.md` → `.claude/skills/*/SKILL.md`
- Added: `.agents/skills` symlink for compatibility
- Removed: `.github/copilot-instructions.md` (redundant)

The newer Claude Code release uses a "skills" structure instead of "agents":
- Each skill has its own directory under `.claude/skills/`
- Skill definition is in `SKILL.md` (instead of a top-level `.md` file)
- Better organization and follows current Claude Code conventions

All 6 agents were migrated:
- deep-research-specialist
- design-pattern-advisor  
- identifier-validator
- metadata-checker
- ontology-reasoner
- task-coordinator

### 4. Documentation: Improved Ontology Editing Guidance
**File**: `CLAUDE.md`

Added/updated critical editing instructions:
- **Reserialization step**: Added requirement to run `robot convert` before commits to ensure proper OBO formatting
- **New term IDs**: Updated prefix from `UBERON:777xxxx` to `UBERON:99xxxxx` (aligning with current ID allocation practices)
- **Removed**: Outdated grep command for checking ID clashes

**File**: `AGENTS.md`
- Created as symlink to CLAUDE.md for agent/skill access
